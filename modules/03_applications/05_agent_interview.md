## A. Agent 原理 🔥🔥🔥

| # | 题目 | 类型 | 热度 | 解题要点 |
|---|------|------|------|----------|
| 1 | Agent 和普通多轮对话的本质区别是什么？ | 原理 | 🔥🔥🔥 | Agent 有自主决策能力、持续状态、目标驱动；对话只是一问一答 |
| 2 | Agent 的核心组件有哪些？缺少哪个影响最大？ | 原理 | 🔥🔥🔥 | 规划、记忆、工具、反思；规划能力是最大瓶颈 |
| 3 | ReAct 的 Thought-Action-Observation 循环在什么情况下会失效？ | 陷阱 | 🔥🔥🔥 | 长任务偏离目标/工具返回错误信息/推理错误级联 |
| 4 | ReAct 和 Plan-and-Execute 的区别？各适合什么场景？ | 对比 | 🔥🔥 | ReAct 适合短任务(边想边做)，P&E 适合长任务(先想后做) |
| 5 | Reflexion 是怎么让 Agent 从错误中学习的？ | 原理 | 🔥🔥 | 执行→评估→反思→存入记忆→下次检索改进 |
| 6 | LATS 将 MCTS 引入 Agent 规划，有什么优缺点？ | 对比 | 🔥 | 优：全局最优；缺：成本高，每个节点都要 LLM 推理 |
| 7 | 层次化规划(Hierarchical Planning)怎么实现？ | 原理 | 🔥🔥 | 高层 Agent 分解子目标，子 Agent 负责执行，类似公司层级 |
| 8 | 动态重规划什么时候触发？和简单重试有什么区别？ | 实战 | 🔥🔥 | 可恢复错误重试，不可恢复错误重规划；重规划改变策略 |

## B. 工具调用 🔥🔥🔥

| # | 题目 | 类型 | 热度 | 解题要点 |
|---|------|------|------|----------|
| 9 | Function Calling 是怎么训练的？模型怎么知道什么时候调工具？ | 原理 | 🔥🔥🔥 | SFT 数据中包含工具调用示例，格式化为特殊 token |
| 10 | Tool Description 写得好与差对结果影响有多大？ | 实战 | 🔥🔥🔥 | 影响巨大，好的描述要有：名称清晰+参数说明+使用示例 |
| 11 | 工具太多(>20个)模型会混淆，怎么解决？ | 实战 | 🔥🔥 | 分类分层/动态工具选择/RAG检索相关工具 |
| 12 | 并行工具调用和串行调用各适合什么场景？ | 对比 | 🔥🔥 | 无依赖关系→并行；有依赖关系→串行 |
| 13 | MCP 和 Function Calling 的本质区别？ | 对比 | 🔥🔥🔥 | FC 紧耦合在应用内，MCP 松耦合可复用跨应用共享 |
| 14 | 工具调用失败时 Agent 应该怎么处理？ | 实战 | 🔥🔥 | 分类错误→重试/换工具/换参数/求助人类 |

## C. 记忆系统 🔥🔥

| # | 题目 | 类型 | 热度 | 解题要点 |
|---|------|------|------|----------|
| 15 | LLM 记忆系统怎么设计？短期/长期/工作记忆分别怎么实现？ | 原理 | 🔥🔥🔥 | 短期=上下文窗口，长期=向量DB/KG，工作=Scratchpad |
| 16 | 上下文窗口满了怎么办？有哪些策略？ | 实战 | 🔥🔥 | 摘要压缩/截断/向量化存储+按需检索 |
| 17 | 记忆的重要性评分怎么设计？ | 实战 | 🔥🔥 | 频率×时效×用户反馈；重要且常用的记忆权重高 |
| 18 | 新旧记忆冲突怎么处理？ | 实战 | 🔥 | 以新信息为准/保留两版+标注时间/请求用户确认 |
