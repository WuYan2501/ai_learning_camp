# 2.1 Transformer 基础 🔥🔥🔥

> 这一节解决一个最核心的问题：**为什么大模型几乎都建立在 Transformer 上。**
>
> 你可以先把 Transformer 理解成一套“让每个 token 在看上下文后，更新自己表示”的机制。它真正厉害的点不只是会看前后文，而是**能高效并行、可扩展、适合做大规模训练**。

## 先建立整体感觉

- **一句话理解**：Transformer 就是在不断重复做两件事：
  - 让 token 彼此“看一眼”，决定谁更重要
  - 再把每个 token 自己做一轮更深入的变换
- **为什么重要**：后面的大模型、长上下文、推理优化、MoE，本质上都还是围着这套骨架做改造
- **先抓主线**：`Attention` → `FFN` → `残差连接` → `归一化` → `位置编码` → `Decoder-only`

---

## Self-Attention（自注意力） ⭐

- **需要懂的内容**：一个 token 为什么能根据上下文动态改变自己的表示
- **简单点理解**：一句话里，不同词彼此的重要程度不一样。Attention 就是在算“当前这个 token 应该重点看谁”
- **数学表达**：`Attention(Q,K,V) = softmax(QKᵀ/√d_k)V`
- **别急着背公式，先知道三件事**：
  - `Q`：我现在想找什么信息
  - `K`：我这里有什么信息可以被别人匹配
  - `V`：真正要被取出来、加权汇总的信息
- **为什么重要**：这一步决定了模型能不能建立长距离依赖、词和词之间的关系、上下文理解能力
- **高频问题**：为什么要除以 `√d_k`？因为维度大了以后点积容易过大，`softmax` 会太尖，训练不稳定

---

## Multi-Head Attention（多头注意力）

- **需要懂的内容**：为什么不是只做一次 Attention，而是做很多个头
- **简单点理解**：一个头可能更关注语法关系，另一个头可能更关注指代关系，还有的头更关心位置或局部模式
- **为什么重要**：多头并行不是为了“更复杂”，而是为了让模型从不同角度同时看上下文
- **常见问题**：不同头学到的东西会不会都一样？理论上可能部分重合，但实践里多头确实能提升表达能力

---

## Feed-Forward Network（FFN） ⭐

- **需要懂的内容**：Attention 让 token 彼此交换信息，那 FFN 在干什么
- **简单点理解**：Attention 更像“先交流一下”，FFN 更像“每个 token 单独再消化一遍信息”
- **为什么重要**：很多人第一次学 Transformer，只盯着 Attention，看不到 FFN 其实也非常重要。实际参数量里，FFN 往往才是大头
- **现代趋势**：早期常见 `ReLU`，现在很多大模型常见 `SwiGLU`
- **常见问题**：为什么 FFN 参数很多？因为它负责更强的非线性变换，可以理解成模型内部的重要“表达能力来源”之一

---

## 残差连接 + 归一化

- **需要懂的内容**：模型为什么能堆这么深，而不是越堆越难训
- **简单点理解**：
  - **残差连接**：给梯度留一条更顺的回路，避免信息传着传着就没了
  - **归一化**：让数值别乱飞，训练过程更稳
- **重点区分**：
  - **Post-Norm**：`LayerNorm(x + Sublayer(x))`
  - **Pre-Norm**：`x + Sublayer(LayerNorm(x))`
- **为什么现代 LLM 更偏向 Pre-Norm**：梯度回传更稳，深层训练更容易

## 位置编码

- **需要懂的内容**：Attention 本身不懂顺序，为什么模型还能知道“谁在前谁在后”
- **简单点理解**：如果不给位置信息，模型只看到一堆 token 的集合，不知道顺序，就很难理解语义
- **主流需要先知道这几种**：
  - **绝对位置编码**：告诉模型“你在第几个位置”
  - **相对位置思想**：更关注“你和我差多远”
  - **RoPE（旋转位置编码）** ⭐：把位置信息编码进 `Q/K`，让注意力天然带相对位置信息
- **为什么现在要优先懂 RoPE**：LLaMA、Qwen、DeepSeek 这一代常见开源模型基本都绕不开它
- **常见问题**：为什么 RoPE 更适合长上下文扩展？因为它更贴近相对位置关系，而不只是死记一个绝对编号

---

## Causal Attention（因果注意力）

- **需要懂的内容**：为什么 GPT 这类模型生成时不能偷看后面的 token
- **简单点理解**：生成下一个 token 时，只能看已经出现的内容，不能先看答案
- **为什么重要**：这决定了 Decoder-only 模型可以做自回归生成，也是今天大模型主流训练方式的基础

---

## 三种经典架构怎么理解

- **Encoder-only**
  - 更擅长“理解输入”
  - 常见于分类、抽取、表示学习
- **Decoder-only** ⭐
  - 更擅长“按顺序生成”
  - 现在的大模型主流基本都属于这一类
- **Encoder-Decoder**
  - 更像“先理解输入，再生成输出”
  - 早期翻译、摘要里很常见

- **为什么今天 Decoder-only 成主流**：
  - 架构更统一
  - 训练目标简单直接
  - 生成任务天然适配
  - 大规模扩展效果更好
